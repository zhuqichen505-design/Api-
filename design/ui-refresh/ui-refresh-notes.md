# Echo UI Refresh Preview Notes

## 目标

本轮先做视觉方向预览，不封包 APK，不改动业务逻辑。预览目标是把 Echo 调整成更精致、简约、适合长时间使用的工具型聊天应用。

## 参考资源提炼

- Material Design 3：使用角色化颜色、清晰层级、稳定组件语义，而不是给每个模块单独发明样式。
- Jetpack Compose Material 3：后续落地时优先通过 `ColorScheme`、`Typography`、`Shape` 和可复用组件统一界面，而不是逐屏硬编码。
- Laws of UX：降低认知负担，重要操作靠位置、留白和层级表达；常用功能可见，低频功能收起。
- Refactoring UI 类资源：少用边框和强阴影，多用间距、字号、灰度、轻量色块建立层级。

## Echo 新界面语言

- 背景：从纯白/浅蓝堆叠改成低噪声雾灰背景，主体内容用白色和极浅蓝灰承载。
- 主色：保留当前蓝色，但降低使用频率，只用于主操作、状态进度和当前选中。
- 强调色：玫红只用于告警、待处理、压缩提醒等语义，不作为大面积装饰。
- 卡片：减少厚重卡片感，使用 12dp 左右圆角、轻边线、几乎无阴影。
- 信息密度：主页、历史、统计保持可扫读；对话页保留更多阅读空间。
- 导航：让滚动辅助、对话导航、上下文状态成为轻量浮层，不打断主任务。
- 文案：按钮和状态提示短而具体，避免说明性长句进入主界面。

## 后续落地建议

1. 先建立统一视觉 token：颜色、圆角、间距、边线、图标尺寸。
2. 再抽出通用组件：顶部栏、筛选芯片、轻卡片、状态条、统计卡片、空状态。
3. 最后逐屏替换：主页 -> 对话页 -> 统计页 -> 设置页 -> 历史/文件夹。
4. 每完成一个界面都做截图验证，避免文字溢出、控件拥挤和状态遮挡。
