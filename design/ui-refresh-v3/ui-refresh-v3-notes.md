# Echo UI Refresh V3 Notes

## 核心约束

V3 的目标不是重新设计 Echo，而是在原版基础上做视觉增强。后续落地时必须遵守：

- 不改功能入口。
- 不改操作路径。
- 不改信息架构。
- 不新增底部导航或新的主导航体系。
- 不把低频功能挪到用户找不到的位置。
- 不改变已有按钮、菜单、弹窗的语义。

## 保留的原版结构

### 主页

- 保留顶部 `Echo` 字标。
- 保留右上角使用统计入口和设置入口。
- 保留搜索栏。
- 保留文件夹筛选区域。
- 保留对话卡片列表。
- 保留右下角“创建对话”按钮。
- 保留长按新建进入配置新对话的逻辑，不在预览中新增替代入口。

### 对话页

- 保留返回按钮。
- 保留标题区和模型/状态提示。
- 保留上下文用量入口。
- 保留对话设置入口。
- 保留系统提示词入口。
- 保留底部输入栏中的附件、图片、OCR、发送等操作入口。
- 主动压缩仍只在上下文弹窗中触发，不放到主输入区。

### 设置页

保留 8 个原菜单入口：

1. API配置
2. 联网搜索
3. 个性化
4. 全局提示词
5. 环境变量
6. 其他对话
7. 数据备份
8. 关于

### 使用统计页

- 保留返回。
- 保留标题和“不准，仅供参考”说明。
- 保留刷新。
- 保留周期切换、总览、Token 消耗、命中率趋势、模型明细。

## 视觉增强范围

- 调整间距和密度，让原界面更顺眼但不改变布局。
- 使用更轻的卡片阴影和边线，提高质感但不显得像网页。
- 保留蓝白清爽基调，少量使用绿色/玫红表达状态。
- 对原有圆形入口、Echo 字标、聊天气泡做细节修饰。
- 统一卡片圆角、按钮尺寸、标签样式和状态颜色。
