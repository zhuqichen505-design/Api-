- 新增 `ConversationContextUsage` 快照模型，用于统一描述当前对话的上下文预算、摘要、记忆和可压缩状态。

## v1.7.3 (2026-07-04) - Token 预算上下文、滚动摘要、长期记忆

### 新增上下文能力
- 新增 token 预算式上下文组装：按模型窗口估算输入预算，并为输出 token 与系统提示保留空间。
- 将上下文拆成近期原文、滚动摘要、长期记忆三类来源，按优先级进入 prompt。
- 替换原先较粗的字符预算策略，减少长对话中旧消息挤占近期消息的问题。

### 新增滚动摘要
- `Conversation` 新增 `rollingSummary`、`summaryUpdatedMessageId`、`summaryUpdatedAt`。
- 当较早对话超过阈值时，自动生成可复用的滚动摘要。
- 摘要生成失败时保留提取式摘要兜底，避免发送请求失败影响主聊天。

### 新增长期记忆表
- 新增 `memory_items` 表，用于保存明确的用户偏好、默认要求、项目背景等长期事实。
- 私密对话不会写入长期记忆。
- 删除对话时同步删除该对话的会话级记忆。
- 聊天请求前会按当前问题与对话范围检索相关记忆，并作为背景注入。

### 数据库与版本
- Room 数据库版本从 16 升到 17。

- `versionName`: 1.7.3

### 验证结果
- release 构建通过。
